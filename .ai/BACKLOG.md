# Initialer Backlog

- [ ] Folien, Dokumente und vorhandene Demo-Artefakte vollständig inventarisieren.
- [ ] Jede fachliche Aussage der Welle 0 gegen aktuelle Primärquellen prüfen.
- [ ] Curriculum-Themen, Folien und geplante Demo-IDs eindeutig zuordnen.
- [ ] Namensschema für Demo-IDs und synthetische Labordaten festlegen.
- [ ] Wiederverwendbares Preflight-, Mess- und Cleanup-Framework entwerfen.

- [ ] Zwei grüne Pilotdemos nach vollständigem Demo-Vertrag umsetzen.

- [ ] Query Store und Extended Events als zentrale Diagnosepfade integrieren.

## P2 - Infrastruktur und Testmatrix

- [ ] SQL-Server-2019/2022/2025-Testmatrix definieren.
- [ ] Docker- und Podman-Kompatibilität der Linux-Labs prüfen.
- [ ] Hyper-V-Grenzfälle und Windows-spezifische Features dokumentieren.
- [ ] Wiederholbare Concurrency-Workloads ohne proprietäre Abhängigkeiten erstellen.

## Erledigungsregel

Ein Punkt gilt nur dann als erledigt, wenn Artefakt, Quellenprüfung und zutreffende Validierung im Repository nachvollziehbar vorhanden sind.
